
## Персональні сповіщення про магнітні бурі

### Ідея

Кожен зареєстрований користувач отримує в особистому кабінеті "скриньку повідомлень" — стрічку персональних сповіщень про наближення сильних магнітних бур з короткими рекомендаціями (100-200 слів). Повідомлення генеруються автоматично ШІ на основі даних NOAA, коли прогнозується підвищена геомагнітна активність (Kp >= 5 або G >= 2).

### Як це працюватиме

1. Коли cron-задача (або існуючий `telegram-forecast`) бачить, що Kp >= 5 або G-шкала >= 2, вона запускає генерацію персональних сповіщень
2. ШІ створює коротке повідомлення (100-200 слів) з описом ситуації та рекомендаціями для здоров'я
3. Повідомлення зберігається в таблицю `user_notifications` для кожного користувача
4. В кабінеті користувач бачить свою скриньку з непрочитаними та прочитаними повідомленнями
5. В хедері з'являється індикатор непрочитаних повідомлень (червона крапка)

### Що побачить користувач

**В хедері** — іконка дзвіночка (або конвертика) поруч з профілем. Якщо є непрочитані — червона крапка з цифрою.

**В кабінеті (`/profile`)** — новий блок "Сповіщення" перед результатами тесту:
- Кожне повідомлення у картці з датою, рівнем бурі (кольоровий бейдж G1-G5), текстом та кнопкою "Прочитано"
- Непрочитані виділені яскравіше
- Можливість позначити всі як прочитані

### Що буде зроблено

**1. Нова таблиця `user_notifications`**

```text
id          uuid (PK)
user_id     uuid (FK -> auth.users, NOT NULL)
title       text
body        text (100-200 слів)
storm_level integer (G-шкала 0-5)
is_read     boolean (default false)
created_at  timestamptz
```

RLS-політики:
- SELECT: користувач бачить тільки свої (`auth.uid() = user_id`)
- UPDATE: може змінити тільки `is_read` своїх записів
- INSERT: тільки через service_role (Edge Function)
- DELETE: заборонено

**2. Edge Function `generate-storm-alerts`**

Нова функція, яку буде викликати cron (або додати логіку в існуючий `telegram-forecast`):
- Перевіряє NOAA дані: якщо Kp >= 5 або G >= 2 — генерує сповіщення
- Використовує Lovable AI (gemini-2.5-flash) для створення тексту 100-200 слів українською
- Текст містить: опис ситуації, рівень бурі, рекомендації (пити воду, уникати стресу, не перевантажуватись)
- Вставляє повідомлення для КОЖНОГО користувача з таблиці `profiles`
- Запобігає дублюванню: не більше 1 сповіщення на день

**3. Блок сповіщень у кабінеті**

Новий компонент `NotificationsInbox` на сторінці `/profile`:
- Список повідомлень (останні 20), відсортованих за датою
- Картка: кольоровий бейдж рівня бурі, заголовок, дата, текст
- Кнопка "Позначити прочитаним" / "Позначити всі прочитаними"
- Порожній стан: "Поки немає сповіщень. Ми повідомимо, коли очікується магнітна буря."

**4. Індикатор непрочитаних у хедері**

Поруч з іконкою профілю — маленька червона крапка з кількістю непрочитаних повідомлень (тільки для залогіненого користувача).

### Логіка генерації

```text
Cron (раз на день о 08:00 Київ)
    |
    v
Перевірка NOAA: Kp >= 5 або G >= 2?
    |
   НІ -> нічого не робити
    |
   ТАК -> Генерація тексту через AI (100-200 слів)
    |
    v
INSERT в user_notifications для всіх user_id з profiles
    |
    v
Готово (опціонально: Web Push "У вас нове сповіщення")
```
